1 Introduction

1.1 Purposes

This is intended to be a really basic Emacs configuration. The main features of this configuration are:

1.2 KISS

Do not include all the super fancy featured packages for IDE or OS-like extension of Emacs, but keep it simple, stupid and thin as possible. It should work as a vanilla Emacs distribution, so trying not to remap main keybindings, but adding a little set of important keychords. This means no Evil mode, but God mode (here's a god article). I somethime miss the mighty vim edit mode, so feel free to activate evil-mode.

1.3 TODO Literate configuration

Basically, what you're reading now is the Emacs configuration: it is an 6 file which will be tangled via Org abel (org-babel-load-file). You only have to copy/paste init.el.

… also take a look to #+PROPERTY: header-args :comments yes in the headers of this file!

1.4 Out-of-the-box

It should be downloaded with CLI tools (like wget, curl), and the only additional operation required in order to let it work is the extraction of the literate part into a dedicated `init.el`. This also means, in some way, that it will be lightweight as possible. If you want to add a particular set of configuration and packages, in order to transform your Emacs into a wonderful IDE on just the unbeatable OS you've ever seen, you can uncomment some blocks you will find along the way.

1.7 Conventions

Somewhere in this configuration file you can find "uncomment the code block". This usually means to remove COMMENT on heading, turn an ORG code block from +begin_example to +begin_src, or to remove the :tangle no attribute.

Maybe is not a best practice for security reasons, but somewhere in this document you can find org-links to elisp code which will run elisp code on the fly, in order to quickly jump somewhere or make something happens in your emacs (very useful for demontration purposes), but is very risky. Ensure org-link-elisp-confirm-function is nil; this way you will be asked for confirmation and here you will see the command displayed in the minibuffer.

Anyway I can’t accept any responsibility or liability for damages arising from evaluating untrusted lisp code on your machine.

5.8 God mode

NOTE: Emacs 25.1 is required for this package to work well.

This is a global minor mode for entering Emacs commands without modifier keys. It's similar to Vim's separation of command mode and insert mode.

All existing key bindings will work in God mode. It's only there to reduce your usage of modifier keys.

God-mode aims to ease keychord immision. You can easily switch it on/off via s-n shortcut. I use took inspiration from vim, so when in god-mode you can just hit i (to deactivate god-mode, something resembling the activation of insert mode in vim) or u for /undo. 

(use-package god-mode
  :config
  (god-mode)
  (which-key-enable-god-mode-support)
  (global-set-key (kbd "s-n") 'god-local-mode)
  (define-key god-local-mode-map (kbd "i") 'god-local-mode)
  (define-key god-local-mode-map (kbd "u") 'undo)
  (define-key god-local-mode-map (kbd "U") 'keyboard-quit)
  (define-key god-local-mode-map (kbd ".") 'repeat)
  (define-key god-local-mode-map (kbd ">") 'end-of-buffer)
  (define-key god-local-mode-map (kbd "<") 'beginning-of-buffer)

  (setq god-exempt-major-modes (append god-exempt-major-modes '(
								elfeed-search-mode
								elfeed-show-mode
								erc-mode
								vterm-mode
								)))

  (defun mm/god-mode-enabled ()
    (setq mm/god-mode-status 1)
    (setq cursor-type 'box))
  (defun mm/god-mode-disabled ()
    (setq mm/god-mode-status nil)
    (setq cursor-type 'bar))
  (add-hook 'god-mode-enabled-hook 'mm/god-mode-enabled)
  (add-hook 'god-mode-disabled-hook 'mm/god-mode-disabled))

(use-package vterm
  :init
  (defun vterm-or-create (&optional last-cmd)
    "Open *vterm* buffer if present, create a new one if none"
    (interactive "p")
    (vterm)
    (when (eq last-cmd 0)
      (progn
        (vterm-send-up)
        (vterm-send-return))))
  (defun vterm-send-last-cmd ()
    (interactive)
    (vterm-send-up)
    (vterm-send-return))

10 Mail

10.1 offlineimap

You can use Emacs as your mail client. In order to configure mu4e I'm following this useful gist. In order to let offlineimap to read your authfile, you must save this snippet into your ~/.offlineimap.py 

#!/usr/bin/python

import re, os

def get_password_emacs(machine, login, port):
    s = "machine %s login %s password ([^ ]*) port %s\n" % (machine, login, port)
    p = re.compile(s)
    authinfo = os.popen("gpg -q --no-tty -d ~/.authinfo.gpg").read()
    return p.search(authinfo).group(1)

and take care to add this entry to your ~/.offlineimaprc 

pythonfile = ~/.offlineimap.py

A normal ~/.authfile.gpg with some entries for IMAP/SMTP account should be like this. 

machine imap.gmail.com login your-email@example.com password your-strong-password port imaps
machine imap.gmail.com login your-email@example.com password your-strong-password port 587
machine smtp.gmail.com login your-email@example.com password your-strong-password port 465

This is an example of an ~/.offlineimaprc config file for a Gmail email account with the inclusion of offlineimap.py defined above. 

[general]
accounts = Gmail
maxsyncaccounts = 1
pythonfile = ~/.offlineimap.py

[Account Gmail]
localrepository = Local
remoterepository = Remote

[Repository Local]
type = Maildir
localfolders = ~/Maildir

[Repository Remote]
type = Gmail
remoteuser = your-email@gmail.com
remotepasseval = get_password_emacs("imap.gmail.com", "your-email@gmail.com", "993")
realdelete = no

folderfilter = lambda foldername: foldername not in ['[Gmail]/Spam', '[Gmail]/All Mail', '[Gmail]/Starred', '[Gmail]/Important']

holdconnectionopen = true
keepalive = 60
sslcacertfile = /etc/ssl/certs/ca-certificates.crt

If you don't have a certificate you can generate via 

sudo mkcert -cert-file /etc/ssl/certs/ca-certificates.crt localhost

(defun mm/yt-download (url &optional audio-only)
  "Download youtube video givin a URL"
  (interactive "sURL: ")
  (let* ((outbuff (format "*yt-download-%s*" url))
	(audio-only (if current-prefix-arg current-prefix-arg 0))
	(audio-only (if (< audio-only 0) "--extract-audio --audio-format mp3" ""))
	(cmd (format "youtube-dl %s \"%s\"" audio-only url))
	)
    (async-shell-command cmd outbuff)))

Now you can invoke the function and a URL will be asked.

  '(default ((t (:height 220)))))

14.7 Opera support

Define browse-url-opera. 

(defun browse-url-opera (url &optional _new-window)
  "Ask the Opera WWW browser to load URL.
Default to the URL around or before point.  The strings in
variable `browse-url-opera-arguments' are also passed to
Opera.
The optional argument NEW-WINDOW is not used."
  (interactive (browse-url-interactive-arg "URL: "))
  (setq url (browse-url-encode-url url))
  (let* ((process-environment (browse-url-process-environment)))
    (apply 'start-process
	   (concat "opera " url) nil
	   browse-url-opera-program
	   (append
	    browse-url-opera-arguments
	    (list url)))))

15.1 Regular Expression Syntax

Here is the syntax used by Emacs for regular expressions. Any character matches itself, except for the list below.

The following characters are special : . * + ? ^ $ \ [

Between brackets [], the following are special : ] - ^

Many characters are special when they follow a backslash – see below.

. any character (but newline)

  • previous character or group, repeated 0 or more time
  • previous character or group, repeated 1 or more time

? previous character or group, repeated 0 or 1 time ^ start of line $ end of line […] any character between brackets [^..] any character not in the brackets [a-z] any character between a and z \ prevents interpretation of following special char \| or \w word constituent \b word boundary \sc character with c syntax (e.g. \s- for whitespace char) \( \) start\end of group \< \> start\end of word \_< \_> start\end of symbol \` \' start\end of buffer\string \1 string matched by the first group \n string matched by the nth group \{3\} previous character or group, repeated 3 times \{3,\} previous character or group, repeated 3 or more times \{3,6\} previous character or group, repeated 3 to 6 times \= match succeeds if it is located at point *?, +?, and ?? are non-greedy versions of *, +, and ? – see NonGreedyRegexp. Also, \W, \B, and \Sc match any character that does not match \w, \b, and \sc.

Characters are organized by category. Use C-u C-x = to display the category of the character under the cursor.

\ca ascii character \Ca non-ascii character (newline included) \cl latin character \cg greek character Here are some syntax classes that can be used between brackets, e.g. [[:upper:]\|[:digit:]\.].

[:digit:] a digit, same as [0-9] [:alpha:] a letter (an alphabetic character) [:alnum:] a letter or adigit (an alphanumeric character () [:upper:] a letter in uppercase [:space:] a whitespace character, as defined by the syntax table [:xdigit:] an hexadecimal digit [:cntrl:] a control character [:ascii:] an ascii character Syntax classes:

\s- whitespace character \s/ character quote character \sw word constituent \s$ paired delimiter \s_ symbol constituent \s' expression prefix \s. punctuation character \s< comment starter \s( open delimiter character \s> comment ender \s) close delimiter character \s! generic comment delimiter \s" string quote character \s| generic string delimiter \s\ escape character

15.2 Vectors

Get an element from a vector 

;; get a element from vector
(aref ["a" "b" "c"] 0) ; ⇒ "a"
;; get a element from vector
(elt ["a" "b" "c"] 0) ; ⇒ "a"

What's the difference between aref and elt? elt is more general. It works on vector and list. But if you know it's vector, you should use aref, because it's precise and faster.

15.3 Timers

You can run specified function at specified time and with a specific timeout! Check out the doc for more information.
